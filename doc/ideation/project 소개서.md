
# Project 소개서 작성 가이드라인

```text
어떤 문제를 해결
어떻게 해결
왜 이 방법이 좋은지
누가 혜택을 받는지
프로젝트의 기대 효과
```

- 소개서 항목
  - 프로젝트명 - 직관적이고 간단하게
  - 시제품 소개 - 한줄로 풀어서 설명(어떤 문제를 어떻게 해결 했는지 볼 수 있게)
  - 상세 내용 - 시제품의 기능과 특성 용도 적용 분야, 키워드, 차별성, 구성 및 구조 등을 명확히 기술
  - 향후 계획 - 내 아이디어가 시장에 진짜 필요한지를검증 하는 항목
  - 시제품 사진 - 기능이 잘 구현되는 사진, 없다면 모델링 사진 AI 사진 가능
  - 심사위원이 가진 의문점을 최대한 해소 할 수 있게 작성

- AI 에 내 제품을 학습
- 제품의 목적, 해결하고자 하는 문제, 주요 사용자, 시장 니즈와 구현하려는 기능, 특성, 용도 ,적용 분야, 키워드, 차별성, 시장조사, 구상 시안

- 페르소나 정하기
- 비즈니스의 핵심 맥락을 명확히 제공하기
- 섹션별로 나눠서 요구하기
- 섹션별로 구체적인 데이터와 키워드 삽입
- 피드백과 수정을 통해 다듬기

- 가독성 좋게 작성해라.
- 신뢰 받는 문서 만들기 - 작은 성과라도 작성, 근거는 수치로, 객관적인 자료를 인용
- 차별화된 부분이 중요 - 도식화, 3D 모델링,
